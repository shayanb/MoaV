8. DNS Tunnel

Last Resort
Port 53/udp

Very slow but works when everything else is blocked. Tunnels data through DNS queries. Only use as last resort.

How DNS Tunnel Works

DNS tunneling encodes data in DNS queries. Since DNS is essential for the internet, it's rarely completely blocked. The trade-off is very slow speeds (expect 10-50 KB/s).

Desktop Setup (Windows/macOS/Linux)

  1. Download dnstt-client for your OS
  2. Open Terminal/Command Prompt
  3. Run this command to create a local SOCKS5 proxy on port 1080:
dnstt-client -doh https://1.1.1.1/dns-query -pubkey {{DNSTT_PUBKEY}} {{DNSTT_DOMAIN}} 127.0.0.1:1080

Then configure your browser to use SOCKS5 proxy: 127.0.0.1:1080

Mobile Setup (Android)

HTTP Injector supports DNS tunneling:

  1. Install HTTP Injector from Play Store
  2. Create new config with DNS Tunnel mode
  3. Enter DNS domain: {{DNSTT_DOMAIN}}
  4. Set remote proxy to your server

Note: iOS has no good DNS tunnel apps. Use desktop if possible.

8b. Slipstream

DNS Tunnel (Faster)
Port 53/udp

QUIC-over-DNS tunnel — 1.5-5x faster than dnstt. Use when other methods are blocked. Requires the Slipstream client and certificate file from this bundle.

Desktop Setup (Windows/macOS/Linux)

  1. Download slipstream-client for your OS
  2. Copy the slipstream-cert.pem file from this bundle to the same directory
  3. Open Terminal/Command Prompt
  4. Run this command to create a local SOCKS5 proxy on port 1080:
slipstream-client --domain {{SLIPSTREAM_DOMAIN}} --cert slipstream-cert.pem --dns-server 1.1.1.1:53 --socks-listen 127.0.0.1:1080

Then configure your browser to use SOCKS5 proxy: 127.0.0.1:1080

Speed Modes

Resolver mode (default, stealthier): ~60 KB/s — good for chat, email, light browsing

Authoritative mode (faster): ~3-4 MB/s — add --authoritative SERVER_IP:53 instead of --dns-server

Authoritative mode is faster but reveals the server IP to network observers.

  2. WireGuard را به صورت عادی وصل کنید

تونل DNS چگونه کار می‌کند

تونل DNS داده‌ها را در درخواست‌های DNS کدگذاری می‌کند. چون DNS برای اینترنت ضروری است، به ندرت کاملاً مسدود می‌شود. سرعت بسیار کم است (انتظار ۱۰-۵۰ کیلوبایت در ثانیه).

۸ب. Slipstream

تونل DNS (سریع‌تر)
پورت 53/udp

تونل QUIC-over-DNS — ۱.۵ تا ۵ برابر سریع‌تر از dnstt. وقتی روش‌های دیگر مسدود هستند استفاده کنید. نیاز به کلاینت Slipstream و فایل گواهی از این بسته دارد.

راه‌اندازی دسکتاپ (Windows/macOS/Linux)

  1. slipstream-client را برای سیستم عامل خود دانلود کنید
  2. فایل slipstream-cert.pem را از این بسته در همان پوشه قرار دهید
  3. Terminal یا Command Prompt را باز کنید
  4. این دستور را اجرا کنید:
slipstream-client --domain {{SLIPSTREAM_DOMAIN}} --cert slipstream-cert.pem --dns-server 1.1.1.1:53 --socks-listen 127.0.0.1:1080

سپس مرورگر را برای استفاده از پروکسی SOCKS5 تنظیم کنید: 127.0.0.1:1080
